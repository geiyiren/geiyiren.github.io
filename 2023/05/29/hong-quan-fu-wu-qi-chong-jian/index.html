权限配置

smb文件共享

  • 安装smb服务

    sudo apt install samba -y

  • 配置共享文件夹

    hopechart@hopechart:~$ cat /etc/samba/smb.conf
[global]
   workgroup = dlsusers
   server string = deep-learning-server share
   dns proxy = no
   log file = /var/log/samba/log.%m
   max log size = 1000
   syslog = 0
   panic action = /usr/share/samba/panic-action %d
   server role = standalone server
   passdb backend = tdbsam
   obey pam restrictions = yes
   unix password sync = yes
   passwd program = /usr/bin/passwd %u
   passwd chat = *Enter\snew\s*\spassword:* %n\n *Retype\snew\s*\spassword:* %n\n *password\supdated\ssuccessfully* .
   pam password change = yes
   map to guest = bad user
   usershare allow guests = no
   # for share directary soft links
   follow symlinks = yes
   wide links = yes
   unix extensions = no
[shares]
   path = /home/shares
   browseable = yes
   writable = yes
   available = yes
   public = yes
   create mask = 0744
   directory mask = 0755
   valid users = @dlsusers

数据盘权限配置

递归配置挂载文件的权限：755

root@delldls:/home/dls# cat auto-chown.sh 
#!/bin/bash

# 设置目标目录
TARGET_DIR="/home/shares"

# 进入目标目录
cd "$TARGET_DIR" || { echo "目录 $TARGET_DIR 不存在！"; exit 1; }
echo "当前目录为 $TARGET_DIR"
# 遍历一级子目录
for dir in */; do
    # 去掉末尾的斜杠
    dirname=${dir%/}

    # 检查用户名是否存在
    if id "$dirname" &>/dev/null; then
        sudo chown -R "$dirname:root" "$dir"
        echo "设置 $dir 属主为 $dirname:root"
    else
        sudo chown -R root:root "$dir"
        echo "设置 $dir 属主为 root:root"
    fi

    # 设置目录权限为 755
    sudo chmod -R 755 "$dir"
done
root@delldls:/home/dls# nohup bash auto-chown.sh > auto-chown.log 2>&1 &

nfs文件共享

客户端

1、安装nfs服务

sudo apt install nfs-kernel-server

2、配置共享文件夹

在/etc/exports文件中增加如下配置：

/home/shares/rknn_toolkit_common *(rw,sync,no_root_squash,no_subtree_check)

3、启动nfs服务

sudo systemctl start nfs-server

服务端

1、安装nfs客户端

sudo apt install nfs-common

2、添加如下配置到~/.bashrc中

count=`ls /rknn_toolkit_common|wc -w`

work_dir=/rknn_toolkit_common

if [ $count -gt 0 ];
then
    echo "already mount:${work_dir}"
else
    echo "mount:${work_dir}"
    mount -t nfs -o nolock 172.20.1.103:/home/shares/rknn_toolkit_common ${work_dir}
fi

win多用户挂载同一目录

win在同一个网络共享路径（UNC 路径，比如 \\server\share），不能同时用多个用户名连接，但是可以修改配置给服务器起别名，方法如下：

  • 1、使用终端管理员打开C:\Windows\System32\drivers\etc\hosts

    notepad C:\Windows\System32\drivers\etc\hosts

  • 2、在文件尾部增加服务器别名，示例如下：

172.20.1.10 hqguest
172.20.1.10 wangshuaiyang

  • 3、映射网络驱动器

    \\wangshuaiyang\shares

    \\hqguest\shares

同时需要保证服务器上该目录允许两个用户访问共享文件，可修改访问者为指定组的所有成员：

[shares]
   path = /home/shares
   browseable = yes
   writable = yes
   available = yes
   public = yes
   create mask = 0744
   directory mask = 0755
   valid users = @dlsusers
   #valid users = sunxing
   #force user = sunxing
   #force group = dlsusers
   comment = smb share for deeplearning

docker安装

安装

1、检查卸载老版本Docker

sudo apt-get remove docker docker-engine docker.io containerd runc

2、更新软件包

# 小凹兔助你快速部署安装Docker
sudo apt-get update
sudo apt-get upgrade

3、安装docker依赖

sudo apt-get install ca-certificates curl gnupg lsb-release

4、添加docker密钥

curl -fsSL http://mirrors.aliyun.com/docker-ce/linux/ubuntu/gpg | sudo apt-key add -

5、添加阿里云docker软件源

sudo add-apt-repository "deb [arch=amd64] http://mirrors.aliyun.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable"

6、安装docker

sudo apt-get install docker-ce docker-ce-cli containerd.io

7.安装nvidia-container-runtime

sudo apt install nvidia-container-runtime

配置

  • 修改docker启动命令

    sudo vi /lib/systemd/system/docker.service

    找到ExecStart关键字，按照如下配置进行修改

    ExecStart=/usr/bin/dockerd --tlsverify --tlscacert=/etc/docker/certs.d/ca-dls.pem --tlscert=/etc/docker/certs.d/server-cert-dls.pem --tlskey=/etc/docker/certs.d/server-key-dls.pem -H unix:///var/run/docker.sock -H tcp://0.0.0.0:2376

  • 修改docker配置文件

    sudo vi /etc/docker/daemon.json

    修改后的内容如下所示：

    {
    "default-runtime": "nvidia",
    "runtimes": {
        "nvidia": {
            "path": "nvidia-container-runtime",
            "runtimeArgs": []
        }
    },
    "insecure-registries":["172.20.1.10:10086"]
}
