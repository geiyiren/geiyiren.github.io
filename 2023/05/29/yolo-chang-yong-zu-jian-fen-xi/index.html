残差网络的核心思想是：每个附加层都应该更容易地包含原始函数作为其元素之一

C3模块分析

ResNet结构分析

首先来看下ResNet的残差结构是什么样子的：

import torch
from torch import nn
from torch.nn import functional as F
from d2l import torch as d2l


class Residual(nn.Module):  #@save
    def __init__(self, input_channels, num_channels,
                 use_1x1conv=False, strides=1):
        super().__init__()
        self.conv1 = nn.Conv2d(input_channels, num_channels,
                               kernel_size=3, padding=1, stride=strides)
        self.conv2 = nn.Conv2d(num_channels, num_channels,
                               kernel_size=3, padding=1)
        if use_1x1conv:
            self.conv3 = nn.Conv2d(input_channels, num_channels,
                                   kernel_size=1, stride=strides)
        else:
            self.conv3 = None
        self.bn1 = nn.BatchNorm2d(num_channels)
        self.bn2 = nn.BatchNorm2d(num_channels)

    def forward(self, X):
        Y = F.relu(self.bn1(self.conv1(X)))
        Y = self.bn2(self.conv2(Y))
        if self.conv3:
            X = self.conv3(X)
        Y += X
        return F.relu(Y)

1x1卷积是为了调整通道和分辨率

  • ResNet的优点

    • 可以有效缓解梯度消失

      残差结构的函数表达式如下所示：
      $$
      y=F(x)+x
      $$
      其中$x$表示模块输入，$F$为虚线框对应的操作，$y$为残差模块的输出，这里将损失表示为$\mathcal{L}$，则进行损失的反向传播时需要计算损失$\mathcal{L}$对输入$x$的导数,由链式法则可知:
      $$
      \frac{\partial\mathcal{L}}{\partial x}=\frac{\partial\mathcal{L}}{\partial y}\cdot\frac{\partial y}{\partial x}
      $$
      则：
      $$
      \frac{\partial y}{\partial x}=\frac{\partial F(x)}{\partial x}+\frac{\partial x}{\partial x}=\frac{\partial F(x)}{\partial x}+I
      \\
      \frac{\partial\mathcal{L}}{\partial x}=\frac{\partial\mathcal{L}}{\partial y}\cdot\left(\frac{\partial F(x)}{\partial x}+I\right)
      $$
      这样当主干分支的梯度$\frac{\partial F(x)}{\partial x}$为0时(出现了梯度消失)，由于$I$为单位矩阵，那么梯度仍旧可以沿着$\frac{\partial\mathcal{L}}{\partial y}$进行传播。

    • 可以缓解深度网络退化现象

      这里的退化可以理解为当网络变深后，如果没有残差结构，梯度即使不消失也会很小，这样浅层收到梯度就会越来越弱，这样网络虽然深了，但是浅层的网络参数无法进行训练了，网络的精度有可能不升反而降低

    • 增加了网络的表达能力

      说人话就是网络可以映射更为复杂的关系，这是因为每个残差模块即可什么都不做($F(x)=0$)也可以做些什么($F(x)\neq0$)，另一个方面由于残差块可以有效缓解梯度消失问题，这也有助于提升网络的深度，进一步增强网络的复杂性，提升表达能力

而DenseNet正是在ResNet的基础上进行了更加稠密的连接:

其中c代表的是channel级连接操作

这样的网络首先具有特征复用充分的特征，后面的每一层网络都可以使用前面任何一层提取的特征，实现了一定程度上的特征共享。而ResNet是将上以模块的提取的特征进行重新学习后形成自己的特征进行表达输出。但是像DenseNet这样的网络不适合深层设计，因为随着层数增加，连接数急剧增长，带来计算复杂度上升，训练非常深的DenseNet变得困难。

ResNet是将主干和旁路分支的结果进行相加后融合为自己的特征进行表达输出，而DenseNet是主干和旁路分支的结果进行拼接后融合为自己的特征进行表达输出，这样可以实现上一个模块的特征未经”加工”且可以在这一模块直接使用。

BottleNeck

层级 卷积核 作用 通道数 特征图尺寸
Conv1 1×1 降维 C_in → C_mid H × W（不变）
Conv2 3×3 特征提取 C_mid → C_mid H/stride × W/stride（可能变）
Conv3 1×1 升维 C_mid → C_out 与上一步相同（不变）

看起来和ResNet有点像，但其实不一样，主要区别如下：

项目 ResNet的残差模块 Bottleneck模块
主体卷积结构 3×3 → 3×3 1×1 → 3×3 → 1×1
通道是否变换 保持一致 中间压缩（降低）再升维
参数量 较少 更多
计算效率 一般 更高（因中间通道更少）
特征图尺寸变换 stride 通常在第1个卷积 stride 通常在中间或最后1个卷积
残差连接 输入通道=输出通道 需要升降维（用1×1卷积调整）
模块结构复杂度 简单 更复杂
表达能力 一般 更强（深而窄）

Bottleneck模块为了中间层卷积的计算量更小，在第一层卷积中将卷积核设置为1x1进行降维，减少通道数，然后使用中间层3x3卷积进行特征提取，最后再使用1x1卷积进行升维度。Bottleneck模块通过降维-提取-升维的设计，既减少计算负担又支持更深的网络，利用更丰富的非线性变换(降维-升维)和特征融合，从而提升了网络捕捉复杂特征的能力，表达能力更强。

C3结构分析

  • C3 模块设计亮点总结：

    1. 残差结构借鉴（ResNet）：通过旁路分支实现跳跃连接，改善深层网络的训练稳定性；

    2. 特征复用（DenseNet 思想）：主干堆叠多个 Bottleneck 模块，实现连续特征共享；

    3. 计算优化（Bottleneck 引入）：使用轻量结构降低模型参数与计算量；

    4. 先降维后提取：在 Bottleneck 前引入 3×3 卷积降维，减少后续计算负担同时也进行了一定程度的局部空间特征提取；

    5. 融合平滑：主干与旁路融合后使用 3×3 卷积进一步整合特征，提升表达一致性。

      • 解释什么是“一致性”：

        1. 语义一致性

        • 融合的两个分支，一个走了 Bottleneck、一个直接跳跃（旁路），它们在语义层次上可能存在差异。
        • 使用 3×3 卷积可以对融合后的特征做“重新整合”，让这些语义差异减小，最终输出的特征更统一、能更好地表达当前区域的语义信息。

        2. 空间一致性

        • 主干分支和旁路分支可能由于卷积、激活、通道数变化等带来不同的空间响应模式。
        • 使用一个 3×3 卷积后，相邻像素之间的特征会更加平滑和连贯，减少“拼接感”或“突兀的边界”。

        3. 统计一致性

        • 两个分支的特征可能具有不同的数值分布（例如均值、方差、通道能量差异），拼接后可能会影响后续网络处理。
        • 3×3 卷积可以视为一种“局部的归一化 + 重映射”操作，使输出特征的统计属性更加一致，便于下游模块处理。

从结构上对比C2f和C3的不同之处：

  • C2f取消了旁路分支的卷积操作。
  • C2f在利用特征提取之前首先按照通道将特征划分为两部分，只有一半参与特征提取，另一半与提取后的特征进行融合。这样的操作在参数量上可以明显下降。之所以不说推理速度可以提升是以为C2f使用了通道划分操作，这个操作在不同的设备上进行推理时优化的方法有所不同，甚至在rknn1.6之前都不支持这一操作。而C3的卷积在不同设备上推理时都支持且都进行了优化。
  • C3具有的优势C2f就基本上都具有，但是C2f相对更加轻量化

SPPF

池化

首先来看下什么是池化，下面是最大池化：

首先可以看出池化后特征图的尺寸变小了:

  • 特征图越小，感受野越大，因此池化可以增加感受野
  • 特征图越小，参数量越小，后面对这个特征图进行卷积操作时计算量就小了
  • 池化属于降采样，本质是对特征图的缩放，这样图像的细节特征就被隐藏了，因此池化还可以在一定程度上增加特征的缩放不变性和平移不变性

上下文信息

上下文信息（Contextual Information），简单来说，就是某个目标、像素或特征所在环境的额外信息，不仅仅看它自己，还要考虑它周围的空间、场景、结构、语义等多方面的背景。

类型 说明 例子
空间上下文 目标周围的空间结构和布局 一辆车的轮廓与车身、车窗和轮胎的相对位置
语义上下文 目标和环境的语义关系 行人在道路上更合理，空中漂浮的“行人”可能是假
全局上下文 整张图像的整体信息 图片是室内还是室外，能影响对物体的理解
时间上下文 视频或连续帧中时间上的连续性 行人在前一帧的位置和运动趋势
  • 池化对上下文信息的影响
    • 池化是对图像的缩放，增加了感受野，那么特征图中的每个值所代表的区域信息就更多了，因此池化后的特征图更加关注图像的全局信息，比如目标容易出现在图像中左侧偏黑，右侧偏白的图像中，也就是说感受野越大，图像的整体结构特征越容易被学习到
    • 类似于均值滤波和最大值滤波，池化也可以减少图像的细节噪声，使得图像的上下文信息更加稳定

SPP

池化的好处这么多，网络中怎么能没有池化呢？首先看下SPP结构怎么使用池化的:

首先将得到的特征进行卷积,且该卷积为1x1的，只是进行降维，默认通道数减半，然后进行不同尺寸的池化，最后进行拼接后再次进行维度变换。其本质是将不同尺度的上下文信息进行融合。并且这样的结构支持任意通道的特征输入和指定通道的特征输出。

SPPF

SPPF是对SPP的改进，实现的功能是类似的,只是计算量和效果略有不同:

  • 用 SPPF 结构，把一个大池化核拆成连续的3次5×5 max-pooling，减少了计算，提升了速度。池化为降采样，串联的池化相对于并联的池化计算量更大，因此SPPF相对于SPP计算量有所降低。

  • 串行的池化比并行的池化更加节省内存

  • SPPF是运行速度比SPP快一些

  • 从多尺度特征融合的角度来看，SPPF的特征融合能力稍微弱于SPP。SPPF每次池化的输入是上一次池化的输出，特征图尺寸逐步缩小，多尺度的细粒度特征逐渐减弱，但感受野逐渐加大，上下文信息逐步丰富

这里有个问题，SPPF或者SPP存在的意义是什么，去掉SPPF层会有什么影响:

方面 可能的影响
多尺度感知能力 可能依赖其他结构保证，失去了骨干网络最后的池化层带来的多尺度融合特征
模型复杂度 降低，结构更简单
推理速度 提升，尤其是对边缘设备友好
准确率 轻微影响或无影响，依赖整体架构调整

C2fCIB

深度可分离卷积和普通卷积

  • 普通卷积

可以看出普通卷积是将各个通道和各自对应的卷积核进行卷积后逐元素相加后组成了新的特征图，且输入通道和输出通道可以不同，卷积核的个数=输出通道数x输入通道数

  • 深度可分离卷积

深度可分离卷积的首要特征就是每个通道使用独立的卷积核，且输入通道数等于输出通道数，计算量大大降低，毫无疑卷积核的个数=输出通道数=输入通道数

深度可分离卷积的明显特征就是计算量大大下降，这是因为卷积核的个数下降，权重参数自然也下降了。

CIB

注意：不同版本中RepVGGDW也可以换为普通的深度可分离卷积

可以看出CIB结构就是对BottleNeck结构的改进，将部分的普通卷积换为了深度可分离卷积，使用深度可分离卷积+1x1卷积构成一个完整的深度可分离卷积结构(DW+PW结构)，同时给出了一个可以替换的RepVGGDW模块:

RepVGGDW 的优点就是可以使用卷积融合方法，就是把训练阶段多条卷积路径和残差，通过权重叠加，合并成一个普通卷积核，保证推理阶段的输出与训练阶段一致，同时大幅提升推理效率。那么为什么可以进行卷积融合呢？

  • 卷积的线性特性

    假设输入特征图为 $X$，卷积核为 $W$，偏置为 $b$，卷积输出：

$$
Y = X * W + b
$$

​ 这是一个线性映射

​ 满足线性运算性质：

​ 加法结合性：
$$
X*(W_1 + W_2) = XW_1 + XW_2
$$
​ 标量分配性：
$$
X*(\alpha W) = \alpha (X*W)
$$
​ 这就意味着多个卷积输出可以直接通过卷积核加和得到等效卷积。

  • 融合多卷积路径

​ 假设训练阶段 RepVGGDW 有两条卷积路径：

​ 主卷积：$X * W_1$

​ 残差卷积：$X * W_2$

​ 输出为：
$$
Y = XW_1 + XW_2
$$
​ 根据卷积的线性特性，可以直接合并为：
$$
Y = X*(W_1 + W_2)
$$

​ 偏置也可以直接相加：
$$
b_{\text{fused}} = b_1 + b_2
$$
那么就可以在训练时使用多分支，增加网络的表达能力在推理时使用单分支不降低任何性能。

C2fCIB

现在再来看C2fCIB模块就很明了：

与C2f结构的不同之处就是，C2fCIB将BottleNeck模块换为了CIB模块，其目的就是为了降低计算量

SCDown

先看结构：

看完这个结构图，有点懵逼，前面使用CIB的时候不是说DW+PW是经典的深度可分离卷积吗？这里怎么又开始PW+DW组合了，这是想怎么组合就怎么组合？那么先来看看ChatGPT怎么说：

特性 DW → PW PW → DW
先提取空间特征 ✔ ✖（通道数先改变，空间卷积受影响）
通道融合时机 PW 在后，跨通道特征融合 PW 在前，先调整通道，后续 DWConv轻量
计算量 随输入通道数增长 随输出通道数增长，可减少计算量
下采样 DWConv 可以 stride=1 或 stride>1 DWConv 通常用于下采样
非线性激活 PWConv 可加激活 PWConv 可加激活，DWConv 保持线性

为什么要换顺序？

  • 根据模块功能不同：
    1. 经典特征提取模块：DW→PW，强调表达能力
      • MobileNet、普通深度可分离卷积模块
    2. 下采样/轻量化模块：PW→DW，强调效率与降低计算量
      • SCDown、RepVGGDW 中的下采样模块
  • 动机：
    • DWConv 的计算量和通道数成正比
    • 先 PW 可以把通道数降低 → 后续 DWConv 更轻量
    • 对于下采样模块，空间尺寸减半后通道已经调整好，更高效
  1. DW→PW：先空间卷积再通道融合 → 表达能力强 → 适合特征提取模块
  2. PW→DW：先通道调整再空间卷积 → 计算量低 → 适合下采样或轻量化模块
  3. 选择顺序的意义：根据模块功能（表达 vs 下采样/轻量化）和计算效率来调整
  4. 核心原则：DWConv 计算量随通道数线性增长 → 改顺序可以控制 FLOPs

也就是说先合并通道数再进行卷积和先进行卷积在合并通道的区别，先合并通道的情况下卷积的计算量会大大降低，那么就比较适合偏向于降采样的操作，因为这样可以降低计算量。如果先进行卷积再进行通道合并就是偏重于特征提取，强调表达能力。那么如果说合并通道处理的是通道特征，那么卷积就是在处理空间特征，那么什么是空间特征，什么是通道特征：

  • 空间特征（Spatial Features）

    • 作用范围：卷积核在二维空间（H×W）上的滑动

    • 处理对象：每个通道内部的局部邻域像素关系

      • 典型意义：
        • 保留局部纹理（edges, corners, patterns）
        • 描述物体的形状轮廓、位置关系
        • 高分辨率阶段的细节捕捉能力强

    • 直观理解：DWConv 就像拿着一块放大镜，在每个颜色通道单独看局部的“花纹”

    • 更偏向：细节信息（local details）

  • 通道特征（Channel Features）

  • 作用范围：跨通道的线性组合（1×1卷积）

  • 处理对象：不同通道之间的特征模式关系

    • 典型意义：
      • 将不同通道的特征重新组合，形成更高级别的语义表示
      • 可以理解为整合来自不同卷积核的“感知结果”
      • 融合后的特征包含更宽的上下文理解能力（比如同时考虑颜色、纹理、位置）

  • 直观理解：PWConv 像是不同相机镜头拍的照片合成一张综合的图

  • 更偏向：上下文信息（context semantics）

再回头分析一下CIB中的深度可分离卷积是没有进行降采样的，而SCDown中的深度可分离卷积是进行了降采样。到这里恍然大悟，原来C3、C2f,C2fCIB都不进行将采样，都是通过卷积进行特征提取的，如果这个卷积是经过设计的那么就可以提取边缘信息比如Sobel算子、Laplacian算子。而深度学习中的卷积的权重信息是在训练的过程中自动调整的，所以会根据损失反向调整权重使其提取想要的高层语义信息。

C3K2

C3K

首先开看一下C3K，这个就比较水了，在C3中Bottleneck 中的卷积核的尺寸固定为3x3，C3K就是Bottleneck的卷积核可指定配置为5x5或7x7等自定义的卷积核。看代码就知道C3K继承自C3,只是重定义了Bottleneck 。

C3K2

这个异曲同工，同样是偷梁换柱，C3K2继承自C3只是配置可以将Bottleneck换为C3k，增加模块的多样性而已。

C2PSA

残差网络的核心思想是：每个附加层都应该更容易地包含原始函数作为其元素之一

C3模块分析

ResNet结构分析

首先来看下ResNet的残差结构是什么样子的：

import torch
from torch import nn
from torch.nn import functional as F
from d2l import torch as d2l


class Residual(nn.Module):  #@save
    def __init__(self, input_channels, num_channels,
                 use_1x1conv=False, strides=1):
        super().__init__()
        self.conv1 = nn.Conv2d(input_channels, num_channels,
                               kernel_size=3, padding=1, stride=strides)
        self.conv2 = nn.Conv2d(num_channels, num_channels,
                               kernel_size=3, padding=1)
        if use_1x1conv:
            self.conv3 = nn.Conv2d(input_channels, num_channels,
                                   kernel_size=1, stride=strides)
        else:
            self.conv3 = None
        self.bn1 = nn.BatchNorm2d(num_channels)
        self.bn2 = nn.BatchNorm2d(num_channels)

    def forward(self, X):
        Y = F.relu(self.bn1(self.conv1(X)))
        Y = self.bn2(self.conv2(Y))
        if self.conv3:
            X = self.conv3(X)
        Y += X
        return F.relu(Y)

1x1卷积是为了调整通道和分辨率

  • ResNet的优点

    • 可以有效缓解梯度消失

      残差结构的函数表达式如下所示：
      $$
      y=F(x)+x
      $$
      其中$x$表示模块输入，$F$为虚线框对应的操作，$y$为残差模块的输出，这里将损失表示为$\mathcal{L}$，则进行损失的反向传播时需要计算损失$\mathcal{L}$对输入$x$的导数,由链式法则可知:
      $$
      \frac{\partial\mathcal{L}}{\partial x}=\frac{\partial\mathcal{L}}{\partial y}\cdot\frac{\partial y}{\partial x}
      $$
      则：
      $$
      \frac{\partial y}{\partial x}=\frac{\partial F(x)}{\partial x}+\frac{\partial x}{\partial x}=\frac{\partial F(x)}{\partial x}+I
      \\
      \frac{\partial\mathcal{L}}{\partial x}=\frac{\partial\mathcal{L}}{\partial y}\cdot\left(\frac{\partial F(x)}{\partial x}+I\right)
      $$
      这样当主干分支的梯度$\frac{\partial F(x)}{\partial x}$为0时(出现了梯度消失)，由于$I$为单位矩阵，那么梯度仍旧可以沿着$\frac{\partial\mathcal{L}}{\partial y}$进行传播。

    • 可以缓解深度网络退化现象

      这里的退化可以理解为当网络变深后，如果没有残差结构，梯度即使不消失也会很小，这样浅层收到梯度就会越来越弱，这样网络虽然深了，但是浅层的网络参数无法进行训练了，网络的精度有可能不升反而降低

    • 增加了网络的表达能力

      说人话就是网络可以映射更为复杂的关系，这是因为每个残差模块即可什么都不做($F(x)=0$)也可以做些什么($F(x)\neq0$)，另一个方面由于残差块可以有效缓解梯度消失问题，这也有助于提升网络的深度，进一步增强网络的复杂性，提升表达能力

而DenseNet正是在ResNet的基础上进行了更加稠密的连接:

这样的网络首先具有特征复用充分的特征，后面的每一层网络都可以使用前面任何一层提取的特征，实现了一定程度上的特征共享。而ResNet是将上以模块的提取的特征进行重新学习后形成自己的特征进行表达输出。但是像DenseNet这样的网络不适合深层设计，因为随着层数增加，连接数急剧增长，带来计算复杂度上升，训练非常深的DenseNet变得困难。

ResNet是将主干和旁路分支的结果进行相加后融合为自己的特征进行表达输出，而DenseNet是主干和旁路分支的结果进行拼接后融合为自己的特征进行表达输出，这样可以实现上一个模块的特征未经”加工”且可以在这一模块直接使用。

BottleNeck

层级 卷积核 作用 通道数 特征图尺寸
Conv1 1×1 降维 C_in → C_mid H × W（不变）
Conv2 3×3 特征提取 C_mid → C_mid H/stride × W/stride（可能变）
Conv3 1×1 升维 C_mid → C_out 与上一步相同（不变）

看起来和ResNet有点像，但其实不一样，主要区别如下：

项目 ResNet的残差模块 Bottleneck模块
主体卷积结构 3×3 → 3×3 1×1 → 3×3 → 1×1
通道是否变换 保持一致 中间压缩（降低）再升维
参数量 较少 更多
计算效率 一般 更高（因中间通道更少）
特征图尺寸变换 stride 通常在第1个卷积 stride 通常在中间或最后1个卷积
残差连接 输入通道=输出通道 需要升降维（用1×1卷积调整）
模块结构复杂度 简单 更复杂
表达能力 一般 更强（深而窄）

Bottleneck模块为了中间层卷积的计算量更小，在第一层卷积中将卷积核设置为1x1进行降维，减少通道数，然后使用中间层3x3卷积进行特征提取，最后再使用1x1卷积进行升维度。Bottleneck模块通过降维-提取-升维的设计，既减少计算负担又支持更深的网络，利用更丰富的非线性变换(降维-升维)和特征融合，从而提升了网络捕捉复杂特征的能力，表达能力更强。

C3结构分析

  • C3 模块设计亮点总结：

    1. 残差结构借鉴（ResNet）：通过旁路分支实现跳跃连接，改善深层网络的训练稳定性；

    2. 特征复用（DenseNet 思想）：主干堆叠多个 Bottleneck 模块，实现连续特征共享；

    3. 计算优化（Bottleneck 引入）：使用轻量结构降低模型参数与计算量；

    4. 先降维后提取：在 Bottleneck 前引入 3×3 卷积降维，减少后续计算负担同时也进行了一定程度的局部空间特征提取；

    5. 融合平滑：主干与旁路融合后使用 3×3 卷积进一步整合特征，提升表达一致性。

      • 解释什么是“一致性”：

        1. 语义一致性

        • 融合的两个分支，一个走了 Bottleneck、一个直接跳跃（旁路），它们在语义层次上可能存在差异。
        • 使用 3×3 卷积可以对融合后的特征做“重新整合”，让这些语义差异减小，最终输出的特征更统一、能更好地表达当前区域的语义信息。

        2. 空间一致性

        • 主干分支和旁路分支可能由于卷积、激活、通道数变化等带来不同的空间响应模式。
        • 使用一个 3×3 卷积后，相邻像素之间的特征会更加平滑和连贯，减少“拼接感”或“突兀的边界”。

        3. 统计一致性

        • 两个分支的特征可能具有不同的数值分布（例如均值、方差、通道能量差异），拼接后可能会影响后续网络处理。
        • 3×3 卷积可以视为一种“局部的归一化 + 重映射”操作，使输出特征的统计属性更加一致，便于下游模块处理。

池化

首先来看下什么是池化，下面是最大池化：

首先可以看出池化后特征图的尺寸变小了:

  • 特征图越小，感受野越大，因此池化可以增加感受野
  • 特征图越小，参数量越小，后面对这个特征图进行卷积操作时计算量就小了
  • 池化属于降采样，本质是对特征图的缩放，这样图像的细节特征就被隐藏了，因此池化还可以在一定程度上增加特征的缩放不变性和平移不变性

上下文信息

上下文信息（Contextual Information），简单来说，就是某个目标、像素或特征所在环境的额外信息，不仅仅看它自己，还要考虑它周围的空间、场景、结构、语义等多方面的背景。

类型 说明 例子
空间上下文 目标周围的空间结构和布局 一辆车的轮廓与车身、车窗和轮胎的相对位置
语义上下文 目标和环境的语义关系 行人在道路上更合理，空中漂浮的“行人”可能是假
全局上下文 整张图像的整体信息 图片是室内还是室外，能影响对物体的理解
时间上下文 视频或连续帧中时间上的连续性 行人在前一帧的位置和运动趋势
  • 池化对上下文信息的影响
    • 池化是对图像的缩放，增加了感受野，那么特征图中的每个值所代表的区域信息就更多了，因此池化后的特征图更加关注图像的全局信息，比如目标容易出现在图像中左侧偏黑，右侧偏白的图像中，也就是说感受野越大，图像的整体结构特征越容易被学习到
    • 类似于均值滤波和最大值滤波，池化也可以减少图像的细节噪声，使得图像的上下文信息更加稳定

SPP

池化的好处这么多，网络中怎么能没有池化呢？首先看下SPP结构怎么使用池化的:

首先将得到的特征进行卷积,且该卷积为1x1的，只是进行降维，默认通道数减半，然后进行不同尺寸的池化，最后进行拼接后再次进行维度变换。其本质是将不同尺度的上下文信息进行融合。并且这样的结构支持任意通道的特征输入和指定通道的特征输出。

SPPF

SPPF是对SPP的改进，实现的功能是类似的,只是计算量和效果略有不同:

  • 用 SPPF 结构，把一个大池化核拆成连续的3次5×5 max-pooling，减少了计算，提升了速度。池化为降采样，串联的池化相对于并联的池化计算量更大，因此SPPF相对于SPP计算量有所降低。

  • 串行的池化比并行的池化更加节省内存

  • SPPF是运行速度比SPP快一些

  • 从多尺度特征融合的角度来看，SPPF的特征融合能力稍微弱于SPP。SPPF每次池化的输入是上一次池化的输出，特征图尺寸逐步缩小，多尺度的细粒度特征逐渐减弱，但感受野逐渐加大，上下文信息逐步丰富

这里有个问题，SPPF或者SPP存在的意义是什么，去掉SPPF层会有什么影响:

方面 可能的影响
多尺度感知能力 可能依赖其他结构保证，失去了骨干网络最后的池化层带来的多尺度融合特征
模型复杂度 降低，结构更简单
推理速度 提升，尤其是对边缘设备友好
准确率 轻微影响或无影响，依赖整体架构调整

C2fCIB

深度可分离卷积和普通卷积

  • 普通卷积

可以看出普通卷积是将各个通道和各自对应的卷积核进行卷积后逐元素相加后组成了新的特征图，且输入通道和输出通道可以不同，卷积核的个数=输出通道数x输入通道数

  • 深度可分离卷积

深度可分离卷积的首要特征就是每个通道使用独立的卷积核，且输入通道数等于输出通道数，计算量大大降低，毫无疑卷积核的个数=输出通道数=输入通道数

深度可分离卷积的明显特征就是计算量大大下降，这是因为卷积核的个数下降，权重参数自然也下降了。

CIB

注意：不同版本中RepVGGDW也可以换为普通的深度可分离卷积

可以看出CIB结构就是对BottleNeck结构的改进，将部分的普通卷积换为了深度可分离卷积，使用深度可分离卷积+1x1卷积构成一个完整的深度可分离卷积结构(DW+PW结构)，同时给出了一个可以替换的RepVGGDW模块:

RepVGGDW 的优点就是可以使用卷积融合方法，就是把训练阶段多条卷积路径和残差，通过权重叠加，合并成一个普通卷积核，保证推理阶段的输出与训练阶段一致，同时大幅提升推理效率。那么为什么可以进行卷积融合呢？

  • 卷积的线性特性

    假设输入特征图为 $X$，卷积核为 $W$，偏置为 $b$，卷积输出：

$$
Y = X * W + b
$$

​ 这是一个线性映射

​ 满足线性运算性质：

​ 加法结合性：
$$
X*(W_1 + W_2) = XW_1 + XW_2
$$
​ 标量分配性：
$$
X*(\alpha W) = \alpha (X*W)
$$
​ 这就意味着多个卷积输出可以直接通过卷积核加和得到等效卷积。

  • 融合多卷积路径

​ 假设训练阶段 RepVGGDW 有两条卷积路径：

​ 主卷积：$X * W_1$

​ 残差卷积：$X * W_2$

​ 输出为：
$$
Y = XW_1 + XW_2
$$
​ 根据卷积的线性特性，可以直接合并为：
$$
Y = X*(W_1 + W_2)
$$

​ 偏置也可以直接相加：
$$
b_{\text{fused}} = b_1 + b_2
$$
那么就可以在训练时使用多分支，增加网络的表达能力在推理时使用单分支不降低任何性能。

SCDown

先看结构：

看完这个结构图，有点懵逼，前面使用CIB的时候不是说DW+PW是经典的深度可分离卷积吗？这里怎么又开始PW+DW组合了，这是想怎么组合就怎么组合？那么先来看看ChatGPT怎么说：

特性 DW → PW PW → DW
先提取空间特征 ✔ ✖（通道数先改变，空间卷积受影响）
通道融合时机 PW 在后，跨通道特征融合 PW 在前，先调整通道，后续 DWConv轻量

非线性激活 PWConv 可加激活 PWConv 可加激活，DWConv 保持线性

为什么要换顺序？

  • 根据模块功能不同：
    1. 经典特征提取模块：DW→PW，强调表达能力
      • MobileNet、普通深度可分离卷积模块
    2. 下采样/轻量化模块：PW→DW，强调效率与降低计算量
      • SCDown、RepVGGDW 中的下采样模块
  • 动机：
    • DWConv 的计算量和通道数成正比
    • 先 PW 可以把通道数降低 → 后续 DWConv 更轻量
    • 对于下采样模块，空间尺寸减半后通道已经调整好，更高效
  1. DW→PW：先空间卷积再通道融合 → 表达能力强 → 适合特征提取模块
  2. PW→DW：先通道调整再空间卷积 → 计算量低 → 适合下采样或轻量化模块
  3. 选择顺序的意义：根据模块功能（表达 vs 下采样/轻量化）和计算效率来调整
  4. 核心原则：DWConv 计算量随通道数线性增长 → 改顺序可以控制 FLOPs

也就是说先合并通道数再进行卷积和先进行卷积在合并通道的区别，先合并通道的情况下卷积的计算量会大大降低，那么就比较适合偏向于降采样的操作，因为这样可以降低计算量。如果先进行卷积再进行通道合并就是偏重于特征提取，强调表达能力。那么如果说合并通道处理的是通道特征，那么卷积就是在处理空间特征，那么什么是空间特征，什么是通道特征：

  • 空间特征（Spatial Features）

    • 作用范围：卷积核在二维空间（H×W）上的滑动

    • 处理对象：每个通道内部的局部邻域像素关系

      • 典型意义：
        • 保留局部纹理（edges, corners, patterns）
        • 描述物体的形状轮廓、位置关系
        • 高分辨率阶段的细节捕捉能力强

    • 直观理解：DWConv 就像拿着一块放大镜，在每个颜色通道单独看局部的“花纹”

    • 更偏向：细节信息（local details）

  • 通道特征（Channel Features）

  • 作用范围：跨通道的线性组合（1×1卷积）

  • 处理对象：不同通道之间的特征模式关系

    • 典型意义：
      • 将不同通道的特征重新组合，形成更高级别的语义表示
      • 可以理解为整合来自不同卷积核的“感知结果”
      • 融合后的特征包含更宽的上下文理解能力（比如同时考虑颜色、纹理、位置）

  • 直观理解：PWConv 像是不同相机镜头拍的照片合成一张综合的图

  • 更偏向：上下文信息（context semantics）

再回头分析一下CIB中的深度可分离卷积是没有进行降采样的，而SCDown中的深度可分离卷积是进行了降采样。到这里恍然大悟，原来C3、C2f,C2fCIB都不进行将采样，都是通过卷积进行特征提取的，如果这个卷积是经过设计的那么就可以提取边缘信息比如Sobel算子、Laplacian算子。而深度学习中的卷积的权重信息是在训练的过程中自动调整的，所以会根据损失反向调整权重使其提取想要的高层语义信息。
