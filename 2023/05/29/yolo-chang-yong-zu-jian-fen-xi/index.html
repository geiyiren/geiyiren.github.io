残差网络的核心思想是：每个附加层都应该更容易地包含原始函数作为其元素之一

C3模块分析

ResNet结构分析

首先来看下ResNet的残差结构是什么样子的：

import torch
from torch import nn
from torch.nn import functional as F
from d2l import torch as d2l


class Residual(nn.Module):  #@save
    def __init__(self, input_channels, num_channels,
                 use_1x1conv=False, strides=1):
        super().__init__()
        self.conv1 = nn.Conv2d(input_channels, num_channels,
                               kernel_size=3, padding=1, stride=strides)
        self.conv2 = nn.Conv2d(num_channels, num_channels,
                               kernel_size=3, padding=1)
        if use_1x1conv:
            self.conv3 = nn.Conv2d(input_channels, num_channels,
                                   kernel_size=1, stride=strides)
        else:
            self.conv3 = None
        self.bn1 = nn.BatchNorm2d(num_channels)
        self.bn2 = nn.BatchNorm2d(num_channels)

    def forward(self, X):
        Y = F.relu(self.bn1(self.conv1(X)))
        Y = self.bn2(self.conv2(Y))
        if self.conv3:
            X = self.conv3(X)
        Y += X
        return F.relu(Y)

1x1卷积是为了调整通道和分辨率

  • ResNet的优点

    • 可以有效缓解梯度消失

      残差结构的函数表达式如下所示：
      $$
      y=F(x)+x
      $$
      其中$x$表示模块输入，$F$为虚线框对应的操作，$y$为残差模块的输出，这里将损失表示为$\mathcal{L}$，则进行损失的反向传播时需要计算损失$\mathcal{L}$对输入$x$的导数,由链式法则可知:
      $$
      \frac{\partial\mathcal{L}}{\partial x}=\frac{\partial\mathcal{L}}{\partial y}\cdot\frac{\partial y}{\partial x}
      $$
      则：
      $$
      \frac{\partial y}{\partial x}=\frac{\partial F(x)}{\partial x}+\frac{\partial x}{\partial x}=\frac{\partial F(x)}{\partial x}+I
      \\
      \frac{\partial\mathcal{L}}{\partial x}=\frac{\partial\mathcal{L}}{\partial y}\cdot\left(\frac{\partial F(x)}{\partial x}+I\right)
      $$
      这样当主干分支的梯度$\frac{\partial F(x)}{\partial x}$为0时(出现了梯度消失)，由于$I$为单位矩阵，那么梯度仍旧可以沿着$\frac{\partial\mathcal{L}}{\partial y}$进行传播。

    • 可以缓解深度网络退化现象

      这里的退化可以理解为当网络变深后，如果没有残差结构，梯度即使不消失也会很小，这样浅层收到梯度就会越来越弱，这样网络虽然深了，但是浅层的网络参数无法进行训练了，网络的精度有可能不升反而降低

    • 增加了网络的表达能力

      说人话就是网络可以映射更为复杂的关系，这是因为每个残差模块即可什么都不做($F(x)=0$)也可以做些什么($F(x)\neq0$)，另一个方面由于残差块可以有效缓解梯度消失问题，这也有助于提升网络的深度，进一步增强网络的复杂性，提升表达能力

其中c代表的是channel级连接操作

这样的网络首先具有特征复用充分的特征，后面的每一层网络都可以使用前面任何一层提取的特征，实现了一定程度上的特征共享。而ResNet是将上以模块的提取的特征进行重新学习后形成自己的特征进行表达输出。但是像DenseNet这样的网络不适合深层设计，因为随着层数增加，连接数急剧增长，带来计算复杂度上升，训练非常深的DenseNet变得困难。

ResNet是将主干和旁路分支的结果进行相加后融合为自己的特征进行表达输出，而DenseNet是主干和旁路分支的结果进行拼接后融合为自己的特征进行表达输出，这样可以实现上一个模块的特征未经”加工”且可以在这一模块直接使用。

BottleNeck

层级 卷积核 作用 通道数 特征图尺寸
Conv1 1×1 降维 C_in → C_mid H × W（不变）
Conv2 3×3 特征提取 C_mid → C_mid H/stride × W/stride（可能变）
Conv3 1×1 升维 C_mid → C_out 与上一步相同（不变）

看起来和ResNet有点像，但其实不一样，主要区别如下：

项目 ResNet的残差模块 Bottleneck模块
主体卷积结构 3×3 → 3×3 1×1 → 3×3 → 1×1
通道是否变换 保持一致 中间压缩（降低）再升维
参数量 较少 更多
计算效率 一般 更高（因中间通道更少）
特征图尺寸变换 stride 通常在第1个卷积 stride 通常在中间或最后1个卷积
残差连接 输入通道=输出通道 需要升降维（用1×1卷积调整）
模块结构复杂度 简单 更复杂
表达能力 一般 更强（深而窄）

Bottleneck模块为了中间层卷积的计算量更小，在第一层卷积中将卷积核设置为1x1进行降维，减少通道数，然后使用中间层3x3卷积进行特征提取，最后再使用1x1卷积进行升维度。Bottleneck模块通过降维-提取-升维的设计，既减少计算负担又支持更深的网络，利用更丰富的非线性变换(降维-升维)和特征融合，从而提升了网络捕捉复杂特征的能力，表达能力更强。

C3结构分析

  • C3 模块设计亮点总结：

    1. 残差结构借鉴（ResNet）：通过旁路分支实现跳跃连接，改善深层网络的训练稳定性；

    2. 特征复用（DenseNet 思想）：主干堆叠多个 Bottleneck 模块，实现连续特征共享；

    3. 计算优化（Bottleneck 引入）：使用轻量结构降低模型参数与计算量；

    4. 先降维后提取：在 Bottleneck 前引入 3×3 卷积降维，减少后续计算负担同时也进行了一定程度的局部空间特征提取；

    5. 融合平滑：主干与旁路融合后使用 3×3 卷积进一步整合特征，提升表达一致性。

      • 解释什么是“一致性”：

        1. 语义一致性

        • 融合的两个分支，一个走了 Bottleneck、一个直接跳跃（旁路），它们在语义层次上可能存在差异。
        • 使用 3×3 卷积可以对融合后的特征做“重新整合”，让这些语义差异减小，最终输出的特征更统一、能更好地表达当前区域的语义信息。

        2. 空间一致性

        • 主干分支和旁路分支可能由于卷积、激活、通道数变化等带来不同的空间响应模式。
        • 使用一个 3×3 卷积后，相邻像素之间的特征会更加平滑和连贯，减少“拼接感”或“突兀的边界”。

        3. 统计一致性

        • 两个分支的特征可能具有不同的数值分布（例如均值、方差、通道能量差异），拼接后可能会影响后续网络处理。
        • 3×3 卷积可以视为一种“局部的归一化 + 重映射”操作，使输出特征的统计属性更加一致，便于下游模块处理。

从结构上对比C2f和C3的不同之处：

  • C2f取消了旁路分支的卷积操作。
  • C2f在利用特征提取之前首先按照通道将特征划分为两部分，只有一半参与特征提取，另一半与提取后的特征进行融合。这样的操作在参数量上可以明显下降。之所以不说推理速度可以提升是以为C2f使用了通道划分操作，这个操作在不同的设备上进行推理时优化的方法有所不同，甚至在rknn1.6之前都不支持这一操作。而C3的卷积在不同设备上推理时都支持且都进行了优化。
