gdb工具使用

  • 配置程序段错误生成文件的路径
    echo "/userdata/core-%e-%p-%t" |  tee /proc/sys/kernel/core_pattern
  • 运行
    gdb adas_app core-%e-%p-%t
  • 常用命令
    • bt 用于查看程序崩溃时的调用栈
    • frame 4 切换到调用栈中的第 4 层帧

查看库或可执行文件的依赖库

readelf -d  <库路径或可执行文件路径>
readelf -p .comment <executable_file>
strings <executable_file> | grep -i gcc

查看可执行文件所依赖的动态库

ldd <fileName>

调整文件系统大小

resize2fs <选项> <设备路径> <大小>

用于调整 ext2/ext3/ext4 文件系统大小 的命令。它可以 增大或减小 文件系统的大小。<大小>可以省略，表示扩展到设备的最大空间

resize2fs /dev/block/by-name/externApp
扩展exapp文件系统的大小为最大空间

以写的方式重新挂载目录

mount -o rw,remount /exapp/

压缩库或可执行文件

aarch64-v01c01-linux-gnu-strip <文件名>

aarch64-v01c01-linux-gnu-strip为交叉编译工具链中的工具，需要和实际的交叉编译工具链对应

编译加速ccache

ccache 的作用是：加速 C/C++ 编译过程，特别是在源代码没有变化时可以大幅减少编译时间

apt install ccache

然后在CMakeLists.txt中配置：

# 启用 ccache
find_program(CCACHE_PROGRAM ccache)
if(CCACHE_PROGRAM)
    message(STATUS "Using ccache: ${CCACHE_PROGRAM}")
    set(CMAKE_C_COMPILER_LAUNCHER "${CCACHE_PROGRAM}")
    set(CMAKE_CXX_COMPILER_LAUNCHER "${CCACHE_PROGRAM}")
endif()

查看板端支持的命令

命令存储位置一般为：

/bin
/usr/bin/
/usr/sbin/

降低串口打印级别

dmesg -n

LEVEL 含义 打印内容
1 KERN_EMERG 紧急情况（系统崩溃）
3 KERN_ERR 错误信息
4 KERN_WARNING 警告信息
5 KERN_NOTICE 正常但重要的信息
6 KERN_INFO 常规信息
7 KERN_DEBUG 所有调试信息（默认）

dmesg -n 1

“ “开头的文件删除

rm \"\ \"\*

使用反斜杠转义空格和引号

网卡IP配置

# IP配置
ifconfig eth1 10.10.150.37 
# 网卡激活
ifconfig eth1 up

设备内存硬件信息查看

sudo dmidecode -t memory

统计指定目录中所有子目录下文件的个数

find ./ -mindepth 1 -maxdepth 1 -type d | while read dir; do echo "$(basename "$dir") $(find "$dir" -maxdepth 1 -type f | wc -l)"; done
